On parse les arguments des requêtes dans les classes filles de Request

RequestFactory est entièrement statique et a la responsabilité de fabriquer la bonne sous-classe en fonction du premier mot.

Possible de faire la même méthode avec paramètres différents pour accepter les différentes classes

* Server reçoit la request, RequestFactory détermine quelle sous-classe de Request appeler selon le mot clé
* dispatch est virtuelle
* handleRequest pour appeler la bonne sous classe (pour STAT, LIST, DEL...).
